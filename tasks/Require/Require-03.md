# 요구사항
UI 기능 테스트 결과에 대한 수정사항
## 세부내용
* 목록의 입력박스 높이 : 목록 내에 진행업무, 예정업무, 비고 마우스로 클릭하면 입력창으로 최초 변경될때 입력참의 높이를 작성된 내용의 양에 따라 동적으로 조정 (현재는 고정 높이라서 내용이 가려지는 경우가 생김)
* 업무작성 시 세부내용 기호 변경 : ㄴ -> - 로 
* 셀확대편집참 : 화면 하단 고정으로 목록 위로 Overlay 되도록 처리
    ** 주간업무보고 목록이 길경우 목록 하단에 생겨서 화면에서 보이지 않음
* 내주간보고작성 > 전주불러오기 : 전주 내용을 그대로 진행업무, 예정업무, 비고사항을 그대로 불러오도록 변경
* 내주간보고작성 목록정렬 순서 : 프로젝트 정렬순서로 목록 정렬
* 보고서취합 : 제출된 보고서의 경우 엑셀다운로드 기능 추가
* 대시보드 : 주차를 선택하는 기능 추가 : 선택 시 해당 주차의 현황보기
* 파트취합현황 : 보고서 취합 현황으로 변경 : 제출된 경우 선택된 보고서취합 보기화면으로 이동

* 보고서취합 : 조회화면 (제출된 보고서 조회) 시 프로젝트명이 값은 행의 프로젝트셀을 rowspan 해서 병합되서 보이도록 처리
* 보고서취합 : 엑셀단운로드시 보고서 취합 목록과 동일한 헝태로 엑셀 다운로드 되도록 수정
* 보고서 진행업무, 예정업무,비고 내용 조회화며네서 시 항목 레벨별 앞쪽 여백 설정
    ** [] 항목은 현상유지 * 항목은 []고 동일하게, - 항목은 현재 * 항목의 현재 여백으로
