# Require-09: 개인 작업 관리 기능

## 개요
팀원이 자신의 업무를 일 단위로 등록·추적·완료 처리할 수 있는 **개인 작업 관리** 기능을 추가한다.
현재 시스템은 주간업무보고(한일/할일)를 주 단위로 작성하는 구조이나, 일상적으로 발생하는 작업을 그때그때 기록하고 진행 상태를 관리할 수단이 없다.
개인 작업 관리를 통해 **일상 작업 메모 → 주간업무 보고서 작성**의 흐름을 자연스럽게 연결한다.

---

## 1. 작업 등록

### 1.1 기본 등록
- 작업 제목을 입력하여 새 작업을 빠르게 등록할 수 있다
- 등록 시 최소한 제목만 입력하면 되고, 나머지 항목은 나중에 채울 수 있다
- 작업에 다음 정보를 선택적으로 입력할 수 있다
  - **프로젝트**: 소속 팀의 프로젝트 목록에서 선택
  - **우선순위**: 높음 / 보통 / 낮음
  - **마감일**: 날짜 선택 (선택사항)
  - **메모**: 자유 텍스트로 상세 내용 기록

### 1.2 빠른 등록
- 작업 목록 상단에 입력란이 항상 보여서 바로 작업을 추가할 수 있다
- Enter 키를 누르면 즉시 등록되고, 입력란이 비워져서 연속 등록이 가능하다

### 1.3 반복 작업
- 매주 또는 매일 반복되는 작업을 한 번 설정하면 자동으로 생성되도록 할 수 있다
- 예: "주간회의 준비"를 매주 월요일마다 자동 생성

---

## 2. 작업 상태 관리

### 2.1 상태 전환
- 작업은 다음 상태로 관리한다: **할일** → **진행중** → **완료**
- 체크박스를 클릭하면 할일 → 완료로 바로 전환된다
- 완료된 작업은 취소선이 그어지고 목록 하단으로 이동한다
- 실수로 완료 처리한 경우 다시 되돌릴 수 있다

### 2.2 작업 수정
- 작업 제목, 프로젝트, 우선순위, 마감일, 메모를 언제든 수정할 수 있다
- 작업 제목을 클릭하면 바로 수정 모드로 전환된다

### 2.3 작업 삭제
- 더 이상 필요 없는 작업을 삭제할 수 있다
- 삭제 전 확인 메시지가 표시된다

---

## 3. 작업 목록 조회

### 3.1 기본 보기
- 내 작업 목록이 기본적으로 **마감일 순서**로 표시된다
- 마감일이 없는 작업은 등록일 기준으로 정렬된다
- 완료된 작업과 미완료 작업이 구분되어 보인다

### 3.2 필터
- 다음 기준으로 작업을 걸러서 볼 수 있다
  - **상태별**: 할일만 / 진행중만 / 완료만 / 전체
  - **프로젝트별**: 특정 프로젝트의 작업만 보기
  - **우선순위별**: 높음 / 보통 / 낮음
  - **기간별**: 오늘 / 이번 주 / 이번 달 / 마감 지난 작업

### 3.3 정렬
- 마감일순, 우선순위순, 등록일순, 프로젝트별로 정렬을 변경할 수 있다
- 드래그앤드롭으로 작업 순서를 직접 바꿀 수 있다

### 3.4 검색
- 작업 제목이나 메모 내용으로 검색할 수 있다

---

## 4. 주간업무 보고서 연동

개인 작업 관리의 핵심 가치는 **일상 기록이 주간보고서로 자연스럽게 이어지는 것**이다.

### 4.1 작업 → 주간업무 반영
- 이번 주에 완료한 작업을 선택하여 주간업무 보고서의 **진행업무(한일)**에 한 번에 넣을 수 있다
- 진행중인 작업을 선택하여 **예정업무(할일)**에 넣을 수 있다
- 반영 시 작업 제목이 업무항목 텍스트로 자동 변환된다
- 프로젝트가 지정된 작업은 해당 프로젝트 행에 자동 배치된다

### 4.2 주간업무 → 작업 생성
- 주간업무 보고서의 예정업무(할일)에서 개인 작업으로 가져올 수 있다
- 전주 보고서의 할일 항목을 이번 주 작업 목록으로 자동 생성할 수 있다

### 4.3 연동 표시
- 주간업무에 반영된 작업은 별도 표시(아이콘 등)로 구분된다
- 어떤 주차 보고서에 반영되었는지 확인할 수 있다

---

## 5. 대시보드 요약

### 5.1 개인 대시보드에 작업 현황 표시
- 오늘 할 작업 수, 마감 임박 작업 수, 이번 주 완료 작업 수가 한눈에 보인다
- 마감일이 지난 미완료 작업이 있으면 눈에 띄게 강조 표시된다

### 5.2 주간 요약
- 이번 주에 등록한 작업 수, 완료한 작업 수를 확인할 수 있다
- 프로젝트별 작업 분포를 간단히 확인할 수 있다

---

## 6. 접근 범위 및 권한

### 6.1 개인 작업은 본인만 조회·관리
- 내 작업 목록은 오직 본인만 볼 수 있다
- 다른 팀원의 작업을 조회하거나 수정할 수 없다

### 6.2 파트장·팀장 조회 (선택사항)
- 파트장은 소속 파트원의 작업 진행 현황(건수 요약)을 조회할 수 있다 (상세 내용은 비공개)
- 팀장은 팀 전체 작업 현황 요약을 조회할 수 있다

---

## 7. 화면 구성

### 7.1 사이드바 메뉴
- "내 작업" 메뉴가 사이드바에 추가된다 (주간업무 메뉴 근처에 배치)

### 7.2 작업 목록 화면
- 상단: 빠른 등록 입력란 + 필터/정렬 버튼
- 중앙: 작업 목록 (체크박스 + 제목 + 프로젝트 태그 + 마감일 + 우선순위 표시)
- 작업 클릭 시: 우측 패널 또는 모달로 상세 정보 표시/편집

### 7.3 주간업무 반영 화면
- 주간업무 작성 화면에 "내 작업에서 가져오기" 버튼이 추가된다
- 버튼 클릭 시 이번 주 완료/진행중 작업 목록이 표시되고, 선택하여 반영할 수 있다

---

## 8. 유의사항

- 개인 작업 데이터는 팀 소속과 연결된다 (다중 팀 소속 시 현재 선택된 팀 기준)
- 기존 주간업무 작성 기능에 영향을 주지 않도록 독립적으로 구현한다
- 작업 등록·수정은 자동저장 방식으로 동작한다 (주간업무 그리드와 동일한 UX)
- 모바일 화면에서도 기본적인 작업 등록·완료 처리가 가능하도록 고려한다
