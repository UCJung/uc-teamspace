# 주간업무보고 시스템 설계 - 시스템 개요 및 기능적 요구사항

## 1. 시스템 개요

### 1.1 배경 및 목적

현재 선행연구개발팀은 Excel 기반으로 주간업무를 관리하고 있다. 팀은 DX 파트와 AX 파트로 구성되어 있으며, 각 팀원이 프로젝트별로 한일(진행업무)과 할일(예정업무)을 주 단위 시트에 작성하고, 파트장이 이를 취합하여 팀장에게 보고하는 구조이다.

이 방식은 다음과 같은 한계를 가진다.

- 주차별 시트가 개별 탭으로 누적되어 이력 관리 및 검색이 어렵다.
- 파트장이 수작업으로 팀원 업무를 취합해야 하며, 병합·정리에 시간이 소요된다.
- 프로젝트별·팀원별 업무 현황을 다양한 관점에서 조회하기 어렵다.
- 동시 편집 시 충돌 가능성이 있고, 작성 형식이 사람마다 달라질 수 있다.

**주간업무보고 시스템**은 이러한 한계를 해결하고, 팀원 개인의 업무 기록부터 파트장·팀장의 취합·조회까지 일관된 프로세스를 제공하는 웹 기반 시스템이다.

**업무 작성 구조** (현행 3단계)

```
[업무항목명]                    ← 대괄호로 표기, 볼드
 * 세부업무내용                  ← * 기호로 표기
   ㄴ 상세작업내용               ← ㄴ 기호로 상세 기술
```

**현행 작성 컬럼**: 파트, 성명, 프로젝트명, 프로젝트코드, 진행업무(한일), 예정업무(할일), 비고 및 이슈

**현행 주차 관리**: 시트명을 날짜(YYYYMMDD)로 하여 주차별 시트를 생성

## 2. 시스템 범위

### 2.1 대상 사용자

| 역할 | 설명 | 주요 활동 |
|------|------|----------|
| 시스템 관리자 | 시스템 전체 설정 관리 | 팀·파트·프로젝트 기준정보 관리 |
| 팀장 | 팀 전체 업무 총괄 | 전체/파트별 업무 조회, 파트 취합보고 조회 |
| 파트장 | 소속 파트 업무 관리 | 파트원 업무 조회, 파트 주간업무 취합 작성 |
| 팀원 | 개인 업무 작성 | 본인 주간업무(한일/할일) 작성 및 관리 |

### 2.2 시스템 구성 범위

본 시스템은 3개의 주요 기능 영역으로 구성된다.

1. **팀 관리** — 조직 구조(팀-파트-팀원) 및 기준정보 관리
2. **프로젝트 관리** — 수행 프로젝트 및 공통업무 프로젝트 관리
3. **주간업무 관리** — 개인 업무 작성, 파트 취합, 팀장 조회

### 3.3 주간업무 관리

#### 3.3.1 개인 주간업무 작성

| ID | 요구사항 | 설명 | 우선순위 |

| WK-001 | 주간업무 생성 | 해당 주차의 개인 주간업무를 생성한다. 주차는 시작일(월요일) 기준으로 자동 산정한다. | 필수 |
| WK-002 | 한일(진행업무) 작성 | 해당 주에 수행한 업무를 프로젝트별로 작성한다. | 필수 |
| WK-003 | 할일(예정업무) 작성 | 다음 주에 수행할 예정인 업무를 프로젝트별로 작성한다. | 필수 |
| WK-004 | 업무 3단계 작성 | 업무를 '업무항목 > 세부업무 > 상세작업'의 3단계로 구조화하여 작성한다. 세부업무 또는 업무항목 단계까지만 작성할 수 있다. | 필수 |
| WK-005 | 비고(이슈) 작성 | 업무항목별로 비고란에 주요 사항, 이슈, 요청사항 등을 기록한다. | 필수 |
| WK-006 | 프로젝트 선택 | 업무항목 작성 시 등록된 프로젝트 목록에서 프로젝트를 선택한다. 프로젝트코드는 자동 연동된다. | 필수 |
| WK-007 | 업무항목 추가/삭제 | 주간업무 내에서 업무항목(행)을 자유롭게 추가·삭제할 수 있다. | 필수 |
| WK-008 | 전주 예정업무 불러오기 | 주간업무 생성 시, 전주의 예정업무(할일)를 이번 주의 진행업무(한일) 초안으로 자동 불러온다. 아래 WK-008 상세 참조. | 필수 |
| WK-009 | 이전 주차 전체 복사 | 이전 주차의 업무 전체(한일+할일+비고)를 현재 주차로 복사하여 이어 작성할 수 있다. | 선택 |
| WK-010 | 임시저장 | 작성 중인 업무를 임시저장할 수 있다. | 필수 |
| WK-011 | 제출(확정) | 작성 완료된 주간업무를 제출하여 파트장이 조회할 수 있도록 한다. | 필수 |
| WK-012 | 내 업무 이력 조회 | 본인이 작성한 주간업무를 주차별로 조회한다. | 필수 |

#### 3.3.1.1 WK-008 전주 예정업무 불러오기 상세

**개요**: 주간업무의 연속성을 보장하기 위해, 전주에 작성한 예정업무(할일)를 이번 주 진행업무(한일)의 초안으로 자동 불러오는 기능이다. 매주 업무를 처음부터 다시 작성하는 번거로움을 줄이고, 계획 대비 실행 추적을 용이하게 한다.

**동작 흐름**

```
전주 (N-1주)                          이번 주 (N주)
┌─────────────────────┐              ┌─────────────────────┐
│ 프로젝트: 가상병원용인  │              │ 프로젝트: 가상병원용인  │
│                     │   자동복사    │                     │
│ 예정업무(할일):       │ ──────────▶ │ 진행업무(한일):       │
│ [시스템 실증]         │              │ [시스템 실증]         │
│ * 운영 모니터링 및 지원│              │ * 운영 모니터링 및 지원│
│   ㄴ 시스템 안정성 점검│              │   ㄴ 시스템 안정성 점검│
│                     │              │                     │
│                     │              │ 예정업무(할일):       │
│                     │              │ (빈 칸 - 직접 작성)   │
└─────────────────────┘              └─────────────────────┘
```

**기능 요구사항**

| 항목 | 설명 |
|------|------|
| 트리거 시점 | 이번 주 주간업무를 **신규 생성**할 때 자동 실행된다. |
| 복사 대상 | 전주 주간업무의 **예정업무(할일)** 전체 (업무항목, 세부업무, 상세작업 포함) |
| 복사 위치 | 이번 주 주간업무의 **진행업무(한일)** 영역 |
| 프로젝트 매핑 | 전주의 프로젝트 연결 정보를 그대로 유지한다. (프로젝트명 + 코드) |
| 업무항목 매핑 | 전주 업무항목 단위로 불러오며, 동일 프로젝트·동일 업무항목은 하나의 행으로 매핑한다. |
| 할일 영역 | 이번 주의 예정업무(할일) 영역은 **빈 상태**로 생성한다. |
| 비고 영역 | 전주의 비고 내용은 불러오지 않는다. (이번 주에 새로 작성) |
| 편집 가능 | 불러온 한일 초안은 자유롭게 수정·삭제·추가할 수 있다. |
| 선택적 실행 | 사용자가 불러오기를 원하지 않는 경우 건너뛸 수 있다. |
| 항목별 선택 | 불러올 업무항목을 개별 선택/해제할 수 있다. (전체 선택 기본) |
| 전주 미작성 시 | 전주 주간업무가 없는 경우 빈 주간업무를 생성하고 알림을 표시한다. |

| 프로젝트 | 자동 설정 | 전주와 동일한 프로젝트가 연결됨 |
| 업무항목 | 자동 설정 | 전주 할일의 업무항목명이 그대로 들어옴 |
| 진행업무(한일) | **초안 채워짐** | 전주 할일 내용이 복사되어 있음 (수정 가능) |
| 예정업무(할일) | **비어 있음** | 사용자가 직접 작성 |
| 비고 및 이슈 | **비어 있음** | 사용자가 직접 작성 |

#### 3.3.2 업무 작성 데이터 구조

업무항목 한 건의 데이터 구조는 다음과 같다.

| 진행업무(한일) | 해당 주에 수행한 내용 | 필수 |
| 예정업무(할일) | 다음 주에 수행할 예정 내용 | 필수 |
| 비고 및 이슈 | 업무항목별 주요 사항, 이슈, 요청사항 | 선택 |

### 6.2 UI 설계 원칙 — 스프레드시트 그리드형 UI

#### 6.2.1 설계 배경

현행 엑셀 기반 업무 작성 방식은 표 형태에서 여러 업무항목을 한눈에 보며 빠르게 입력할 수 있다는 장점이 있다. 카드형(폼 기반) UI로 전환할 경우 업무항목마다 개별 화면을 오가야 하므로 오히려 작성 생산성이 저하된다. 따라서 본 시스템의 주간업무 작성·조회 화면은 **엑셀과 유사한 스프레드시트 그리드(표) 형태**를 기본 UI로 채택한다.

#### 6.2.2 그리드 UI 기능 요구사항

#### 6.4.3 구조화 서식 자동 적용

진행업무, 예정업무 셀에서 텍스트 입력 시 자동으로 시각적 서식이 적용된다.

```
입력 텍스트                              화면 렌더링

[과제 수주 및 관리]              →       [과제 수주 및 관리]          ← 볼드, 강조색
*정량적 목표 세부사항 정리       →        • 정량적 목표 세부사항 정리   ← 1단 들여쓰기
ㄴ과제 통합 관리 템플릿 작성     →          ㄴ 과제 통합 관리 템플릿 작성 ← 2단 들여쓰기, 회색
```

| 입력 패턴 | 인식 규칙 | 렌더링 서식 |
|----------|----------|-----------|
| `[텍스트]` | 대괄호로 감싼 텍스트 | 볼드체, 강조색(진한 남색) |
| `*텍스트` | 줄 시작이 `*` | 불릿(•) 변환, 1단 들여쓰기 |
| `ㄴ텍스트` | 줄 시작이 `ㄴ` | 2단 들여쓰기, 연한 회색 텍스트 |

## 8. 용어 정의

| 용어 | 정의 |
|------|------|
| 팀 | 조직의 최상위 단위. 하위에 파트를 구성한다. |
| 파트 | 팀 하위의 조직 단위. (예: DX 파트, AX 파트) |
| 파트장 | 파트를 관리하며, 파트원의 업무를 취합하여 보고하는 역할. |
| 프로젝트 | 업무를 수행하는 단위. 프로젝트명과 프로젝트코드로 식별한다. |
| 공통업무 프로젝트 | 조직 운영, 과제 관리 등 팀/파트 공통 업무를 위한 프로젝트. |
| 업무수행 프로젝트 | 실제 연구개발 과제 수행을 위한 프로젝트. |
| 주간업무 | 팀원이 주 단위로 작성하는 업무 기록. 한일과 할일로 구성된다. |
| 한일(진행업무) | 해당 주에 수행한 업무 내용. |
| 할일(예정업무) | 다음 주에 수행할 예정인 업무 내용. |
| 업무항목 | 업무 작성의 1단계. 프로젝트 내 주요 업무 단위. |
| 세부업무 | 업무 작성의 2단계. 업무항목 하위의 구체적 업무. |
